## 第二章：核心业务流程与推理逻辑

### 2.1 数据采集

#### 竞彩网彩池 (5个)

| 彩池 | 字段 | 说明 |
|------|------|------|
| had | h/d/a | 标准胜平负 SP |
| hhad | h/d/a + goalLine | 让球胜平负 SP + 让球数 |
| ttg | s0~s7 | 总进球数 (0~7+) |
| crs | s{hh}{aa}s{hh}{aa} | 正确比分 |
| hafu | hh/hd/ha/dh/... | 半全场 |

### 2.2 推理模型

系统基于泊松分布假设的统计模拟，非机器学习模型。

**核心假设**: 进球数 ~ Poisson(λ)，主客队独立。

#### λ计算

**普通模式**:
```
λ_h = home_xG × away_xGA / 联赛平均xGA
λ_a = away_xG × home_xGA / 联赛平均xGA
```

**世界杯模式** (calc_initial_lambda_wc):
- 应用联合会汇率折算 (CONFED_DEFLATION)
- 支持中立场地
- 使用联合会默认 xGA (CONFED_DEFAULT_XGA)

#### 因子修正

```
λ_h_final = λ_h × (1+injury) × (1+motivation) × (1+altitude)
```

世界杯扩展: INJURY_WORLD_CUP_MAX=0.35, KEY_PLAYER_INJURY_BOOST=0.10, 中立场地处理, GROUP_MATCHDAY_FACTOR, KNOCKOUT_LAMBDA_FACTOR

#### 蒙特卡洛模拟 → DDI校准 → 贴合度评分 → 评级

详见代码: monte_carlo.py / ddi.py / fit_score.py / rating.py

#### 赛前预测模式 (squad_power)

当比赛数据中缺少xG/SofaScore统计数据时（如世界杯开幕前数据冻结阶段），系统自动降级至赛前预测模式：

**数据来源**: data/team_profiles.json (48支参赛队赛前档案)

**核心逻辑**:
`
实力分 = 0.4 × rank_score + 0.6 × value_score
rank_score = 1 - (FIFA排名 - 1) / 49   [归一化至0~1]
value_score = 身价(€M) / 1350M        [归一化至0~1]

λ_h = base_rate × (实力分_h / avg实力分)
λ_a = base_rate × (实力分_a / avg实力分)
`

**联合会惩罚**: 对CONCACAF(0.78)/CAF(0.72)/AFC(0.75)等弱联合会球队实力分乘以CONFED_DEFLATION系数

**触发条件**: 
un_all.py自动检测：若全部4项xG字段均为None且match为世界杯类型，自动调用squad_power.estimate_match()，无需人工干预。

详见: squad_power.py

## 第四章：精选计划单

### 4.1 系统架构

```
竞彩网API → raw_jczq.json → enrich_odds.py → match_info.json
                                         ↓
                              plan_engine.py
                                         ↓
                       API (/api/plan/2.0, /api/plan/3.0)
                                         ↓
                              plan.html 页面
```

### 4.2 核心逻辑

- **单关**: 仅总进球/比分/半全场 (非SP玩法)
- **腿池**: 单选项 + 复选打包 + 混合串
- **串关**: 2串1主推 + 3串1参考
- **评分**: score = model_prob × (fit_score/10) × (1+edge)
- **赔率范围**: 2.0计划 [2.0, 3.0], 3.0计划 [3.0, 4.0]

详见 精选计划单.md
