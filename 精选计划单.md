# 精选计划单系统 — 完整设计 (Rev1.15)

## 一、数据流

```
竞彩网 API (webapi.sporttery.cn)  →  fetch_jczq.py
raw_jczq.json  ← 5个彩池原始数据 (had/hhad/ttg/crs/hafu)
  ↓ enrich_odds.py
match_info.json  ← 注入 jc_sp_*, jc_hhad_*, ttg_odds, crs_odds, hafu_odds
locked_data.json ← 同上
  │
  ├── monte_carlo_result.json  ← MC模拟 (lambda → 概率)
  ├── rating_result.json       ← 贴合度评分 + 评级
  │
  └── plan_engine.py
        ↓
  API: /api/plan/2.0, /api/plan/3.0  →  plan.html 渲染
```

auto_pipeline.py 全流程跑完后自动执行 fetch + enrich。

## 二、赔率来源

| 彩池 | Key 格式 | 示例 |
|------|---------|------|
| 胜平负 (SP) | had.h/d/a | 1.15, 3.55 |
| 让球胜平负 | hhad.h/d/a + goalLine | 2.25, 1.89 |
| 总进球数 | ttg.s0~s7 | s0=18.0, s3=3.6 |
| 正确比分 | crs.s{Home}{Away}s{Home}{Away} | s00s00=10.0 |
| 半全场 | hafu.hh/hd/ha/dh/... | hh=1.65, aa=2.1 |

## 三、评分公式

```
edge = model_prob - 1/odds          ← 模型相对市场的优势
score = model_prob × (fit_score / 10) × (1 + edge)
```

- model_prob：MC物理概率（SP）或泊松分布（进球/比分）或市场反推（半全场）
- fit_score：贴合度评分，低于 6.0 或 meltdown 跳过
- score 用于所有排序

## 四、策略规则

### 4.1 单关（独立投注）

- 仅考虑：总进球数、比分、半全场（非SP玩法）
- 赔率范围：2.0计划 [2.0, 3.0]，3.0计划 [3.0, 4.0]
- 按 score 降序取 TopN

### 4.2 腿池（串关构建块）

**A. 单选项腿** — 所有玩法、score > 0 的选项直接作为腿。低赔(odds<1.20)需 EV≥1.03

**B. 复选打包腿** — 同比赛同玩法 2-3 选项打包：
```
虚拟赔率 = 1 / (1/odds1 + 1/odds2 + ...)
score = 各子选项 score 均值
```

**C. 混合串腿** — 同比赛不同玩法直接相乘：
```
赔率 = odds_A × odds_B
score = (score_A + score_B) / 2
```
取全局 Top 8 进入腿池。

### 4.3 跨比赛串关

- **2串1（主推）**: 不同比赛的两条腿，乘积赔率在范围内，按 score 均值降序 Top10
- **3串1（参考）**: 三条腿，三个不同比赛，乘积赔率在范围内

## 七、当前局限

1. **3串1 输出量大**，目前只取 Top10
2. **复选打包 score 用均值**，未考虑选项间相关性
3. **混合串** 取全局 Top8，竞彩规则对同比赛多玩法串关的限制需确认
