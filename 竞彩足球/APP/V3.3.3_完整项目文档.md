## 一、项目概述

**V3.3.3** 是一个 2026 世界杯足球比赛 AI 预测分析系统。前端是一个手机端单页面应用，数据由独立预测 AI agent 写入 JSON 文件驱动。

### 技术栈
- 前端：纯 HTML + CSS + JavaScript（单页，无框架）
- 数据：3 个 JSON 文件（`analysis.json` / `config.json` / `news.json`）
- 服务端：Python 3 自带的 `http.server`（仅本地开发/局域网用）
- 部署：GitHub Pages（https://athomas8285.github.io/worldcup2026/）
- 数据推送：Git commit & push

### 核心原则
1. **数据驱动** — 所有动态内容从 JSON 加载，不写死在 HTML 中
2. **预测 agent 只改 JSON**，不改 HTML/CSS/JS
3. **前端无依赖** — 不需要 Node.js、npm、构建工具

## 三、页面结构（index.html 前端）

### 单页面应用设计
`index.html` 是唯一的 HTML 文件。所有页面（首页、分析、资讯、更多）都在同一个 HTML 中，通过 `display:none` / `display:flex` 切换显示。

### 页面布局
```
┌─────────────────────┐
│  顶部导航栏 (Header) │  — 标题 + 连接状态指示灯
├─────────────────────┤
│                     │
│  页面内容区域       │  — 四个页面之一：
│  (滚动)            │    pageHome / page02 / page03 / page04
│                     │
├─────────────────────┤
│  底部导航栏 (Tab)    │  — 首页 / 分析 / 资讯 / 更多
└─────────────────────┘
```

### 四个页面

| 页面 ID | Tab 名 | 功能 |
|---------|--------|------|
| `pageHome` | 🏔 首页 | 倒计时、最近比赛预测、今日赛程切换 |
| `page02` | 🔳 分析 | 日期栏 + 比赛预测卡片（可展开） |
| `page03` | 📰 资讯 | 置顶文章 + 文章列表 |
| `page04` | ⚙️ 更多 | 数据源状态、使用引导、作者信息、页脚 |

### 底部 Tab 切换

`switchTab(id, el)` 函数控制页面切换：
1. 隐藏所有 `.page` 元素
2. 显示目标页面
3. 移除所有 `.tab-item` 的 `active` 类
4. 给点击的 tab 加 `active`
5. 特殊处理：切换到 page02 时调用 `renderMatches(today)` 渲染当天比赛

### 加载流程（启动顺序）

```
1. HTML 加载
2. CSS 渲染（深色背景 + 背景图）
3. loadingOverlay 显示（进度条动画 + "正在加载数据，请稍等几秒钟"）
4. JS 执行 → 定义所有函数（renderMatches, buildDateBar, renderStand 等）
5. initApp() 异步执行：
   a. fetch('data/analysis.json')  — 解析为 __DATA, AL, GR, DN
   b. fetch('data/config.json')     — 解析为 _cfg
   c. renderCountdown()             — 首页倒计时
   d. renderSettings(cfg)           — 更多页配置
   e. fetch('data/news.json')       — 解析为新闻数据
   f. renderNews()                  — 资讯页文章
   g. buildDateBar()                — 分析页日期栏
   h. 高亮今天的日期
   i. renderMatches(今天)           — 分析页当天比赛
   j. loadingOverlay 隐藏          — 页面显示完成
```

## 四、前端核心函数说明

### 数据全局变量（initApp 加载后可用）
| 变量 | 来源 | 类型 | 用途 |
|------|------|------|------|
| `__DATA` | analysis.json | `{ rating, mc, narratives }` | 比赛预测、模型计算、分析文本 |
| `AL` | analysis.json | `{ "MM-DD": [matchId, ...] }` | 日期→比赛ID 映射 |
| `GR` | analysis.json | `{ "A": [team, ...] }` | 小组分组 |
| `DN` | analysis.json | `{ "MM-DD": "周X" }` | 日期→星期对照 |
| `_cfg` | config.json | object | 旗帜、场馆、数据源、引导等 |

### 渲染函数

| 函数 | 用途 | 数据来源 |
|------|------|----------|
| `renderMatches(dateKey)` | 渲染指定日期的比赛卡片 | `__DATA.rating` + `__DATA.mc` |
| `renderCountdown()` | 首页倒计时（下一场最近比赛） | `__DATA.rating` |
| `buildDateBar()` | 构建日期选择栏 | `AL`, `DN` |
| `renderSchedule()` | 渲染全部赛程表 | `WM`（硬编码，待优化） |
| `renderStand()` | 渲染小组积分榜 | `__DATA.rating`（有 actual_score 的） |
| `renderNews(news)` | 渲染置顶文章 + 文章列表 | `news.json` |
| `renderSettings(cfg)` | 渲染更多页的配置信息 | `config.json` |

### 工具函数

| 函数 | 用途 |
|------|------|
| `switchTab(id, el)` | 切换页面 Tab |
| `selectDate(dateKey, el)` | 选择日期（日期栏点击） |
| `dirClass(direction)` | CSS 类：direction→胜/平/负样式 |
| `rateClass(rating)` | CSS 类：评级 A/B/C 样式 |
| `fl(name)` | 球队名→国旗 emoji |
| `ucd()` | 首页倒计时更新（每秒） |
| `openArticle(id)` | 打开文章弹窗 |
| `showQR()` / `closeQR(e)` | 二维码弹窗 |

### 比赛卡片渲染逻辑（renderMatches）

每张比赛卡片有三种状态：
```
if (有 actual_score) → 赛后复盘卡片（折叠式，显示命中/未中）
else if (无 direction 或无 mc.physical) → "等待预测数据" 提示
else → 预测卡片（展开显示方向、概率、拟合分等）
```

## 七、Bug 修复历史 & 关键决策

### 已全部修复
1. **页面不加载（"JS_STARTED" 卡住）**
   - 原因：page02 合并后残留了 `buildDateBar()` 直接调用，在 `initApp()` 之前执行，`AL` 为空报错
   - 修复：删除两个多余的 `buildDateBar()` 立即调用，它们已在 `initApp()` 末尾正常执行

2. **分析页不显示 JSON 数据**
   - 原因：page02 的脚本里有两个 `var __DATA={...}` 硬编码数据副本
   - 修复：改为 `var __DATA=null`，由 `initApp()` 从 JSON fetch 后赋值

3. **预测日期硬编码为 "pending"**
   - 原因：`renderMatches` 里写了 `var isPending=["06-15","06-16","06-17","06-18"]`
   - 修复：改为按单场比赛是否有 `direction` 和 `mc.physical` 动态判断

4. **小组积分榜数据不对**
   - 原因：`renderStand()` 用的 `WM` 硬编码数组，不是 JSON 数据
   - 修复：改为遍历 `__DATA.rating` 中有 `actual_score` 的条目计算积分

5. **"今天" 日期写死 06-14**
   - 原因：`buildDateBar()` 中 `var t = k === "06-14"`，`initApp()` 中 `indexOf(14)>=0`
   - 修复：改为 `new Date()` 动态计算当天日期

6. **导航点击无响应**
   - 原因：页面元素缺少正确的 `id` 和 `switchTab` 绑定
   - 修复：统一页面 ID 和 tab 点击事件

7. **滚动失效**
   - 原因：CSS 中 `display:block` 覆盖了 `display:flex`
   - 修复：保留 `.page { display:flex }`

8. **文章 `???` 占位符**
   - 原因：硬编码文章卡片中未替换中文
   - 修复：由 `renderNews()` 从 JSON 动态渲染

### 硬编码残留（暂时不管）
- `WM` 数组：`renderSchedule()` 仍在使用，但 `renderStand()` 已改为 JSON
- `FL` 对象：5 份国旗 emoji 映射（应从 config.json 生成）
- `GR` 对象：2 份副本（虽然 initApp 会覆盖）
- 重复函数：5 组函数各有 2-3 份定义（不会报错但文件臃肿）
- 硬编码文章卡片：在 `newsSection` 中，被 JS 覆盖

## 十、更新工作流

### 更新预测数据
1. 预测 agent 跑完数据采集和模型计算
2. 告诉用户数据已更新
3. 用户（或 agent）执行 `git add` + `commit` + `push`
4. GitHub Pages 自动同步

### 数据更新规则
- `analysis.json.rating[]` 和 `mc[]`：按 `id` 查找，存在就覆盖，不存在就 push
- `config.json`：有新增球队或比赛时同步添加
- `news.json.articles[]`：**不自动追加**，由用户手动指示后操作

### 关键注意事项
- 球队名在 JSON 文件中必须**完全一致**（大小写、繁简体）
- 比赛 ID 在所有 JSON 中必须一致
- 新球队必须先在 `config.json.teamFlags` 中添加旗帜
- 新比赛 ID 必须先在 `config.json.venues` 中添加场馆
- 赛后 5 个字段一起填：`actual_score` + `hit` + `result` + `score` + `predClass`
- 所有 JSON 文件使用 **UTF-8 无 BOM** 编码

## 十二、本地服务器（serve_utf8.py）

```python
import http.server, socketserver, io, os
os.chdir(os.path.dirname(os.path.abspath(__file__)))
class H(http.server.SimpleHTTPRequestHandler):
    def send_head(self):
        p = self.translate_path(self.path)
        try:
            with open(p, "rb") as f: d = f.read()
        except: return self.send_error(404)
        ct = self.guess_type(p)
        if ct == "text/html": ct = "text/html; charset=utf-8"
        self.send_response(200)
        self.send_header("Content-Type", ct)
        self.send_header("Cache-Control", "no-cache, no-store, must-revalidate")
        self.send_header("Content-Length", len(d))
        self.end_headers()
        return io.BytesIO(d)
with socketserver.TCPServer(("", 8898), H) as httpd:
    print("Serving on 8898")
    httpd.serve_forever()
```

特点：设置了 UTF-8 编码、禁用缓存、从文件所在目录提供服务。
