# V3.3.3 数据契约

## 概述

本系统由三个 JSON 文件驱动前端渲染。以下定义每个文件的字段结构、类型约束和填写规则。

---

## 1. `analysis.json` — 比赛与分析数据

```json
{
  "rating": [ ... ],
  "mc": [ ... ],
  "narratives": { ... },
  "AL": { ... },
  "GR": { ... },
  "DN": { ... }
}
```

### `rating` — 球队实力评级
- 类型: `array`，每个元素为 `object`
- 字段:
  - `name`: `string` — 球队名称（必须与 `GR` 和 `AL` 引用一致）
  - `rating`: `number` — 实力评分
  - `rank`: `number` — 排名（1-48）
- 示例:
  ```json
  { "name": "巴西", "rating": 95.2, "rank": 1 }
  ```

### `mc` — 比赛计算数据
- 类型: `array`，每个元素对应一场比赛
- 字段:
  | 字段 | 类型 | 说明 |
  |------|------|------|
  | `id` | `string` | 比赛 ID，格式如 `周四001`（与 venues 键一致） |
  | `home` | `string` | 主队名称 |
  | `away` | `string` | 客队名称 |
  | `lambda_raw_h` | `number` | 原始主队 lambda |
  | `lambda_raw_a` | `number` | 原始客队 lambda |
  | `lambda_h_final` | `number` | 调整后主队 lambda |
  | `lambda_a_final` | `number` | 调整后客队 lambda |
  | `lambda_diff` | `number` | lambda 差值 |
  | `jc_handicap` | `number` | 竞彩让球数 |
  | `physical` | `object` | 物理模型概率 `{ home_win, draw, away_win }` |
  | `jc_handicap_prob` | `object` | 竞彩让球概率 `{ rang_sheng, rang_ping, rang_fu }` |
  | `top2_total_goals` | `array` | 推荐总进球前 2 选项，如 `["2球","3球"]` |
  | `top2_half_full` | `array` | 推荐半全场前 2 选项，如 `["胜胜","平胜"]` |
  | `top3_scores` | `array` | 推荐比分前 3 选项，如 `["2-0","3-0","1-0"]` |

### `narratives` — 文字分析
- 类型: `object`，key 为比赛 ID
- 值类型: `string` — 分析文字
- 示例:
  ```json
  { "周四001": "模型主胜 87.2% / 差+2.33，赔率趋势自6月8日起连续6次压低主胜..." }
  ```

### `AL` — 赛程列表
- 类型: `object`，key 为日期 `"MM-DD"`
- 值类型: `array` of `string` — 比赛 ID 列表
- 示例:
  ```json
  { "06-11": ["周四001","周四002"] }
  ```

### `GR` — 小组分组
- 类型: `object`，key 为小组名 `"A"` ~ `"L"`
- 值类型: `array` of `string` — 球队名称
- 球队名称必须全局一致（与 `rating`、`AL` 引用统一）
- 示例:
  ```json
  { "A": ["墨西哥","捷克","韩国","南非"] }
  ```

### `DN` — 日期星期映射
- 类型: `object`，key 为日期 `"MM-DD"`
- 值类型: `string` — 星期简称
- 示例:
  ```json
  { "06-11": "周四", "06-12": "周五" }
  ```

## 2. `config.json` — 配置数据

```json
{
  "dataSources": [ ... ],
  "guide": [ ... ],
  "teamFlags": { ... },
  "venues": { ... },
  "author": { ... },
  "footer": "..."
}
```

### `dataSources` — 数据源
- 类型: `array`
- 每个元素字段:
  - `name`: `string` — 数据源名称（如 "China Jingcai"）
  - `status`: `string` — `"on"` 或 `"off"`
  - `desc`: `string` — 简短说明

### `guide` — 使用引导
- 类型: `array`
- 每个元素字段:
  - `icon`: `string` — emoji 图标名（如 "recycle", "chart", "hourglass"）
  - `title`: `string` — 标题
  - `desc`: `string` — 描述文字

### `teamFlags` — 球队旗帜与颜色
- 类型: `object`，key 为球队名称（必须与 analysis.json 一致）
- 每个值:
  - `flag`: `string` — 国旗 emoji
  - `color`: `string` — CSS 渐变色（如 `"linear-gradient(135deg,#555,#555)"`）

### `venues` — 场馆映射
- 类型: `object`，key 为比赛 ID
- 每个值:
  - `group`: `string` — 小组名
  - `venue`: `string` — 比赛场馆城市

### `author` — 作者信息
- 类型: `object`
- 字段:
  - `bio`: `string` — 作者简介
  - `contactLabel`: `string` — 联系方式标签

### `footer` — 页脚
- 类型: `string` — 支持换行符 `\n`

## 3. `news.json` — 资讯数据

```json
{
  "pinned": { ... },
  "articles": [ ... ]
}
```

### `pinned` — 置顶文章
- 类型: `object`
- 字段:
  | 字段 | 类型 | 说明 |
  |------|------|------|
  | `title` | `string` | 文章标题 |
  | `date` | `string` | 日期，格式 `YYYY-MM-DD` |
  | `source` | `string` | 来源名称 |
  | `views` | `number` | 阅读量 |
  | `sections` | `array` | 章节列表 |
- 每个 `sections` 元素:
  - `heading`: `string` — 小标题
  - `body`: `string` — 正文

### `articles` — 普通文章列表
- 类型: `array`
- 每个元素字段:
  | 字段 | 类型 | 说明 |
  |------|------|------|
  | `title` | `string` | 文章标题 |
  | `date` | `string` | 日期，格式 `YYYY-MM-DD` |
  | `summary` | `string` | 摘要（80 字以内为宜） |
  | `url` | `string` | 跳转链接（必填，不可为 `"#"`） |
  | `views` | `number` | 阅读量 |

---

## 4. 填写规则

### 名称一致性
- **球队名称** 必须在所有 JSON 文件中保持一致：
  - `analysis.json` 的 `GR`、`mc[].home`、`mc[].away`
  - `config.json` 的 `teamFlags` 的 key
- 名称不一致会导致旗帜不显示、积分榜计算错误。

### 比赛 ID 一致性
- 比赛 ID（如 `"周四001"`）必须在以下位置一致：
  - `analysis.json` 的 `mc[].id`、`AL` 数组元素、`narratives` 的 key
  - `config.json` 的 `venues` 的 key
- 格式示例：`周四001`、`周五003`

### 日期格式
- `AL` 和 `DN` 的 key 使用 `"MM-DD"` 格式（如 `"06-14"`）
- `articles[].date` 和 `pinned.date` 使用 `"YYYY-MM-DD"` 格式

### 小组命名
- 小组名为 `"A"` ~ `"L"`（2026 世界杯 12 组）
- 每组 4 支球队

### 编码
- 所有 JSON 文件必须使用 **UTF-8 编码，无 BOM**
- 包含中文和 emoji，请勿转义为 Unicode 序列

### 链接
- `articles[].url` 必须是真实可访问的 URL
- 不可为 `"#"` 或空字符串
- 推荐前缀: `https://worldcup2026cn.com/news/`
