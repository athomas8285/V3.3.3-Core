## 一、项目概览

世界杯竞彩预测系统。每天 11:00 竞彩网开盘后，对当天销售的一批比赛做分析预测，产出方向和评级，同步到两个前端页面。

### 两个前端
| 版本 | 技术 | 线上地址 | 本地目录 |
|------|------|---------|---------|
| 网页版（Flask） | Flask + SQLite | https://web-production-5a133.up.railway.app/ | D:\V3.3.3-Core |
| APP 版（手机） | 静态页面 | https://athomas8285.github.io/worldcup2026/ | C:\Users\gjj\Documents\Codex\2026-06-14\app-app\outputs |

### 数据源
| 数据源 | 用途 | 访问方式 |
|--------|------|---------|
| 竞彩网 API (webapi.sporttery.cn) | 当前在售比赛、5彩池赔率 | requests + User-Agent + Accept-Language: zh-CN |
| 500.com (zx.500.com) | 历史赛果（比分+半全场） | requests + GBK encoding |
| SofaScore (api.sofascore.com) | 近期战绩、赛季数据（可能被墙） | Playwright（备选） |

### 核心数据流
```
竞彩网 11:00 开盘
  ↓ (fetch_jczq.py + enrich_odds.py)
framework.db（SQLite，唯一数据源头）
  ↓ (auto_pipeline.py / run_all.py)
rating_result.json + match_info.json
  ↓ (生成 analysis.json)
APP 端数据 → GitHub Pages
```

## 三、数据采集（原 Agent A）

### 3.1 抓取竞彩网当天开盘比赛
```powershell
cd D:\V3.3.3-Core
python fetch_jczq.py
```
- 从竞彩网 API 拉取所有在售比赛数据
- 5 彩池：胜平负、让球胜平负、总进球、比分、半全场
- 输出：data/raw_jczq.json

### 3.2 赔率数据注入
```powershell
python enrich_odds.py
```
- 将 raw_jczq 同步到 match_info.json 和 locked_data.json
- 覆盖：亚洲盘口、赔率

### 3.3 合并写入数据库
通过 database.py 写入 framework.db，新增的插入，已有的跳过。

### 3.4 补录赛果
对 DB 中已完赛但缺 actual_score 的比赛：
1. 从竞彩网历史数据获取比分（比分 + 半全场）
2. 写入 DB 并同步更新 match_info.json 和 rating_result.json
3. 获取不到则标记"赛果待人工"

### 3.5 数据验证
检查当天需要预测的比赛是否在 DB 中都有完整数据。
当天要预测哪些比赛 → 看编号前缀。如当天是周日，则预测"周日XXX"。
核对 match_info.json 中这些比赛是否有赔率数据。

## 四、预测分析（原 Agent B）

### 4.1 判断当天需要预测的比赛
查看 match_info.json 中的比赛编号前缀：
- 周日 → 当天是周日，预测周日037-040 此类
- 周一 → 当天是周一，预测周一041-044 此类
在 DB 中查这些比赛是否已有 direction（已被预测过），已有的跳过。

### 4.2 运行预测管道
```powershell
cd D:\V3.3.3-Core
python auto_pipeline.py
```
或分步运行：
```powershell
python run_all.py
```

管道流程：lambda 计算 → 因子修正 → 蒙特卡洛模拟 → DDI校准 → 贴合度评分 → 评级判定

### 4.3 验证预测结果
检查 `data/rating_result.json`：
- 当天所有待预测比赛都有 direction 和 rating
- 无异常值（fit_score=0 或 direction 为空的需要重跑）

### 4.4 同步预测结果
预测结果同步写入 framework.db（direction, rating, fit_score 等字段）
也同步到 match_info.json 供 APP 同步时使用。

## 五、复盘（原 Agent B）

对前一天的预测结果做复盘诊断：
```powershell
cd D:\V3.3.3-Core
python review.py
```

复盘模块会做：
1. 从 review.json 读取比赛结果
2. 判断每个预测是否命中（hit）
3. 对未命中的做 4 层诊断：数据层/物理层/因子层/市场层
4. 更新 history.csv 和 DB
5. 输出命中率统计

核心指标：累计命中率、本轮命中率、本轮 vs 累计的偏差。

### 复选打包腿规则
将同一场比赛、同一玩法类型的 2-3 个选项打包成一条虚拟腿：
- 虚拟赔率 = 1 / (1/赔率1 + 1/赔率2)
- 虚拟概率 = 各选项概率之和
- 评分 = 各选项 score 均值（score = model_prob x fit/10），只考虑模型概率和贴合度，不考虑赔率

### 6 种打包类型（V3 规则）
| 打包类型 | 具体组合 | 适用场景 |
|---------|---------|---------|
| 总进球打包 | 0/1球 ~ 6/7球(所有相邻对) | 覆盖任意进球区间 |
| 半全场打包 | 主不败(胜胜+平胜)、客不败(负负+平负) | 覆盖不败场景 |
| 比分打包 | 主小胜(1-0+2-0)、客小胜(0-1+0-2) | 覆盖小胜格局 |
| 方向打包(让球盘) | 让胜/平、让平/负 | 覆盖让球盘不败/不胜 |
| 方向打包权重 | 评分 ×1.3（历史命中率60%） | 提升方向打包在串关中的排位 |
| 比分单腿 | ❌ 已移除（仅保留打包） | 半全场单腿保留，比分只做打包 |
